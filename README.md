# to-agents

多智能体通信约定 —— 定义智能体之间的通信方式、消息格式与交互协议。

## 背景

随着 LLM（大语言模型）能力的提升，多智能体协作场景日益普遍。不同框架、不同厂商的智能体之间缺乏统一的通信约定，导致：

- **互操作性差**：不同框架构建的 Agent 难以直接对话
- **协议碎片化**：各家定义各家的消息格式，集成成本高
- **生态割裂**：缺少类似 HTTP 之于 Web 的通用通信层

to-agents 旨在定义一套简洁、可扩展、与框架无关的多智能体通信约定，让不同来源的智能体能够像 REST API 一样自然地交互。

## 设计目标

- **框架无关**：不绑定任何特定的 Agent 框架（LangChain、AutoGen、CrewAI 等）
- **传输无关**：约定层面独立于具体传输层（HTTP、WebSocket、gRPC、stdio 等均可承载）
- **渐进式**：简单场景开箱即用，复杂场景可按需扩展
- **可演化**：版本化协议设计，支持向前兼容
- **中文优先**：文档和讨论以中文为主，同时提供英文版本

## 项目现状

🚧 **早期规划阶段** —— 目前正在定义核心概念和通信模型。

计划涵盖的内容：

- 消息格式规范（Message Schema）
- 能力发现机制（Capability Discovery）
- 会话与上下文管理（Session & Context）
- 流式通信约定（Streaming）
- 错误处理与状态码（Error Handling）
- 安全与认证（Security & Auth）
